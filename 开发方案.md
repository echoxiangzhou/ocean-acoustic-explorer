# OceanAcoustic Explorer — 开发方案

## Context

构建独立的海洋声学交互式可视化平台，不与 OceanAgent 集成，借鉴其 xpublish-wms 瓦片服务方案。数据从头下载处理，5个模块统一架构规划，Docker Compose 部署到私有服务器。声速默认 TEOS-10 (gsw)。

---

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React 18 + TypeScript + CesiumJS + Deck.gl + Plotly.js |
| API 网关 | FastAPI + Uvicorn |
| 瓦片服务 | xpublish + xpublish-wms（WMS 1.3.0 瓦片服务） |
| 声学计算 | Python (arlpy/Bellhop + pyram + gsw) |
| 数据层 | xarray + Zarr |
| 异步任务 | Celery + Redis (broker) |
| 缓存 | Redis |
| 存储 | MinIO (S3 兼容, 存放 Zarr Store) |
| 部署 | Docker Compose + Nginx |

## 核心架构：xpublish-wms 瓦片服务

借鉴 OceanAgent 的方案，用 xpublish + xpublish-wms 将 Zarr 数据暴露为标准 OGC WMS 服务：

```
┌──────────────────────────────────────────────────────────┐
│  Frontend (React + CesiumJS)                              │
│  CesiumJS WebMapServiceImageryProvider 消费 WMS GetMap    │
│  Plotly.js 绘制剖面图/TL图/趋势图                        │
└───────────┬──────────────────────────────┬────────────────┘
            │ WMS GetMap                    │ REST API
            ▼                              ▼
┌───────────────────────┐     ┌──────────────────────────┐
│  xpublish-wms 服务    │     │  FastAPI 后端            │
│  (port 8090)          │     │  (port 8000)             │
│  - 加载 Zarr datasets │     │  - 声速剖面查询          │
│  - OGC WMS 1.3.0      │     │  - 声学计算调度          │
│  - 动态色阶渲染       │     │  - 趋势数据查询          │
│  - 支持 time/depth    │     │  - 场景模拟              │
│    维度切片           │     │  - WebSocket 进度推送     │
└───────────┬───────────┘     └──────────┬───────────────┘
            │                            │
            ▼                            ▼
┌──────────────────────────────────────────────────────────┐
│  MinIO (S3)  ←  Zarr Store                               │
│  - woa23/sound_speed.zarr                                │
│  - features/{channel_axis,surface_duct,...}.zarr          │
│  - gebco/bathymetry.zarr                                 │
│  - soda/trends.zarr                                      │
└──────────────────────────────────────────────────────────┘
```

**WMS 调用示例** (模块A全球图层):
```
GET /wms?service=WMS&request=GetMap
  &layers=channel_axis          # 声道轴深度图层
  &bbox=-180,-90,180,90
  &width=512&height=256
  &crs=EPSG:4326
  &format=image/png
  &transparent=true
  &colormap=viridis             # 色阶
  &vmin=0&vmax=1500             # 值域
  &time=2024-03-01              # 月份 (WMS time 维度)
```

**CesiumJS 前端消费**:
```typescript
viewer.imageryLayers.addImageryProvider(
  new Cesium.WebMapServiceImageryProvider({
    url: '/wms',
    layers: 'channel_axis',
    parameters: {
      format: 'image/png',
      transparent: true,
      colormap: 'viridis',
      vmin: 0, vmax: 1500,
      time: '2024-03-01'
    }
  })
)
```

## 数据获取与预处理

### 数据源

| 数据集 | 用途 | 分辨率 | 获取 | 大小 |
|--------|------|--------|------|------|
| WOA23 T/S | 气候态声速 (模块A/B) | 0.25° × 57层 × 12月 | NOAA HTTP | ~8 GB |
| GEBCO 2023 | 海底地形 | 15弧秒 | HTTP | ~8 GB |
| HYCOM GOFS 3.1 | 实时/准实时 T/S (模块B/C/E) | 0.08° × 40层 | THREDDS 下载 + OPeNDAP 补充 | ~200 GB (近2年全球日均) |
| SODA 3.15.2 | 历史趋势 (模块D) | 0.5° × 50层 × **5天** | HTTP | ~820 GB (2920文件) |
| deck41 | 底质参数 (模块C) | 点数据 36401站点 | HTTP | Shapefile/CSV |

### 预处理流水线

```
1. 下载: WOA23 T/S NetCDF, GEBCO, SODA, HYCOM (近2年全球日均)

2. 转 Zarr:
   WOA23 → Zarr (chunks: {lat:90, lon:90, depth:57, month:1})
   GEBCO → Zarr (chunks: {lat:1000, lon:1000})
   SODA  → Zarr (chunks: {lat:45, lon:45, depth:50, time:12})
   HYCOM → Zarr (chunks: {lat:100, lon:100, depth:40, time:1})

3. 预计算声速场 (TEOS-10 gsw):
   WOA23 T/S → gsw.sound_speed() → sound_speed.zarr

4. 预计算声场特征 (模块A的6个图层):
   对每个 (lat, lon, month):
     - 声道轴深度、表面声道厚度、跃层梯度
     - 会聚区距离、声影区距离 (含 src_depth 维度)
     - 声场类型分类
   → features/*.zarr

5. 预计算趋势 (模块D):
   SODA → 逐月声速 → 特征时间序列 → trends.zarr
```

### Zarr 存储结构 (MinIO)

```
ocean-acoustic/
├── woa23/{temperature,salinity,sound_speed}.zarr
├── features/{channel_axis,surface_duct,thermocline_grad,
│             convergence_zone,shadow_zone,field_type}.zarr
├── gebco/bathymetry.zarr
├── soda/trends.zarr
└── hycom/
    ├── analysis.zarr           # 近2年全球日均 T/S (预下载)
    └── cache/                  # OPeNDAP 按需缓存 (更早/更新数据)
```

## 模块开发方案

### 模块 A：全球声场特征总览

**前端**:
- CesiumJS 3D 地球 + `WebMapServiceImageryProvider` 加载 xpublish-wms 图层
- 6 个图层切换：每个图层 = 一个 WMS layers 参数，切换即更换 imagery provider
- 控件：月份滑块 → 更新 WMS `time` 参数；声源深度滑块 → 更新 `depth` 参数；频率选择器
- Hover tooltip: 调用后端 API 获取该点 6 项特征值（或前端缓存）
- Click → 路由到模块 C

**xpublish-wms**: 直接将 features/*.zarr 注册为 WMS datasets，无需自写渲染逻辑

### 模块 B：声速剖面浏览器

**前端**:
- 左: CesiumJS 地图选点
- 右: Plotly 声速剖面 c(z)（x=声速, y=深度倒置）
- 多剖面对比 (max 5), Plotly shapes 标注声道轴/表面声道/跃层
- 月份动画, 数据源切换 (WOA23/HYCOM)
- 特征值卡片, 声速公式切换 (TEOS-10/Mackenzie/Chen-Millero)

**后端**: `GET /api/profiles?lat=&lon=&month=&source=woa23&formula=teos10`
- WOA23: Zarr 读 T/S → gsw 计算 → 返回 {depth[], sound_speed[], features{}}
- HYCOM: 优先从本地 Zarr 读取 → 缺失时 OPeNDAP 按需获取 → gsw 计算 → Redis 缓存

### 模块 C：区域精细声场分析

**前端**:
- CesiumJS 画线（两点定义断面）或预置断面列表
- 参数面板：声源深度/频率/模型(Bellhop/RAM/Kraken)/距离范围/底质
- WebSocket 接收计算进度
- 结果 tabs:
  - TL(r,z) 伪彩色: Plotly heatmap + 地形轮廓
  - 声线图: Plotly scatter lines
  - 断面声速场: Plotly heatmap
  - 定深 TL 曲线: 可拖拽选深度
  - 定距 TL 曲线: 可拖拽选距离

**后端**:
- `POST /api/acoustic/compute` → Celery 异步任务
- Worker: 插值 T/S → 构建声速场 → GEBCO 地形 → Bellhop/RAM 计算 → TL(r,z) 矩阵
- `WS /api/acoustic/progress/{task_id}` 进度推送
- `GET /api/acoustic/result/{task_id}` 结果获取
- Docker 镜像预编译 Bellhop Fortran 二进制

### 模块 D：时变趋势分析

**前端**:
- 地图选点 / 预定义区域
- Plotly 4子图时间序列 + 线性拟合 + 置信区间
- 可叠加 ENSO/SST 指数 (双Y轴)

**后端**: `GET /api/trends?lat=&lon=` → SODA Zarr 读取 + scipy linregress

### 模块 E：场景模拟器

**前端**: 场景配置表单 → 结果面板 (探测距离/最优深度/CZ方案) → 对比模式
**后端**: 复用模块C计算引擎 + `POST /api/scenarios/evaluate` 声呐方程反推

## Docker Compose 服务

```yaml
services:
  nginx:          # 反向代理 (前端 + 后端 + wms)
    ports: ["80:80"]

  frontend:       # React Vite (开发) / Nginx 静态 (生产)
    ports: ["5173:5173"]

  backend:        # FastAPI 主服务
    ports: ["8000:8000"]
    depends_on: [redis, minio]

  wms-server:     # xpublish-wms 瓦片服务
    ports: ["8090:8090"]
    depends_on: [minio]
    volumes: [zarr_data:/data]

  worker:         # Celery 声学计算 worker (含 Bellhop)
    depends_on: [redis, minio]

  redis:          # 缓存 + Celery broker
    ports: ["6379:6379"]

  minio:          # Zarr 对象存储
    ports: ["9000:9000", "9001:9001"]
```

**Nginx 路由**:
```
/              → frontend
/api/          → backend:8000
/wms           → wms-server:8090
```

## 当前任务：数据下载脚本

目标目录：`ocean-server:/data/nas_data/ocean_acoustic/`

### 脚本清单

1. **download_woa23.sh** — 下载 WOA23 温度和盐度 (0.25° 月平均)
   - 来源: https://www.ncei.noaa.gov/thredds-ocean/fileServer/woa23/DATA/
   - 温度: `temperature/netcdf/decav91C0/0.25/woa23_decav91C0_t{01-16}_04.nc` (16个深度范围文件)
   - 盐度: `salinity/netcdf/decav91C0/0.25/woa23_decav91C0_s{01-16}_04.nc`
   - 存储: `/data/nas_data/ocean_acoustic/woa23/`

2. **download_gebco.sh** — 下载 GEBCO 2024 全球海底地形
   - 来源: https://www.bodc.ac.uk/data/open_download/gebco/gebco_2024/zip/
   - 存储: `/data/nas_data/ocean_acoustic/gebco/`

3. **download_soda.sh** — 下载 SODA 3.15.2 月平均 (1980-2022)
   - 来源: https://dsrs.atmos.umd.edu/DATA/soda3.15.2/REGRIDED/ocean/
   - 文件: `soda3.15.2_mn_ocean_reg_{year}_{month}.nc`
   - 存储: `/data/nas_data/ocean_acoustic/soda/`

4. **download_deck41.sh** — 下载 deck41 海底底质
   - 来源: NOAA NGDC
   - 存储: `/data/nas_data/ocean_acoustic/deck41/`

5. **download_hycom.sh** — 下载 HYCOM GOFS 3.1 全球日均 T/S
   - 来源: https://tds.hycom.org/thredds/ (GLBy0.08 expt_93.0)
   - 范围: 近2年全球日均 (water_temp + salinity)
   - 采用 OPeNDAP NCO 子集下载，按月组织文件
   - 存储: `/data/nas_data/ocean_acoustic/hycom/`

6. **download_all.sh** — 主控脚本，顺序调用上述脚本

### 服务器环境
- OS: Kylin OS (Linux 4.19.90)
- Python: 3.7.9
- 工具: wget, curl
- 存储: NAS 91T (31T 可用)
